## 🎯 О проекте

**Hair Lab** — полнофункциональная система бронирования для барбершопа, включающая:

- 🌐 **Клиентский интерфейс** — современный веб-сайт с возможностью онлайн-бронирования
- 🔐 **Система авторизации** — вход по номеру телефона с кодом подтверждения
- 📅 **Управление бронированиями** — создание, просмотр и отмена записей
- 👨‍💼 **Административная панель** — управление пользователями, проектами и бронированиями
- 📱 **Адаптивный дизайн** — корректная работа на всех устройствах

## 🛠 Технологический стек

### Frontend
- **Next.js 16** — React фреймворк с App Router

## 🚀 Установка и запуск

### Быстрый старт (Windows)

Самый простой способ — использовать автоматический скрипт:

```bash
# Дважды кликните на файл start-all.bat
# Или выполните в командной строке:
start-all.bat
```

Этот скрипт автоматически запустит и бэкенд, и фронтенд.

### Ручная установка

#### 1. Клонирование репозитория

```bash
git clone https://github.com/Sidikov213/Barbershop_hairlab.git
cd Barbershop_hairlab
```

#### 2. Настройка Backend

```bash
# Перейдите в папку backend
cd backend

# Создайте виртуальное окружение
python -m venv venv

# Активируйте виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt

# (Опционально) Инициализируйте БД с тестовыми данными
python init_db.py

# Запустите сервер
python main.py
```

Backend будет доступен по адресу: **http://localhost:8000**

#### 3. Настройка Frontend

Откройте новый терминал:

```bash
# Вернитесь в корень проекта
cd ..

# Установите зависимости
npm install

# Создайте файл .env.local (если его нет)
# Добавьте следующую строку:
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000

# Запустите сервер разработки
npm run dev
```

Frontend будет доступен по адресу: **http://localhost:3000**

## 📚 API Документация

После запуска бэкенда, документация API доступна по адресам:

- **Swagger UI**: http://localhost:8000/docs
- **ReDoc**: http://localhost:8000/redoc

### Основные endpoints

#### Авторизация
- `POST /api/auth/login` — Вход по телефону и коду
- `GET /api/auth/me` — Информация о текущем пользователе
- `PUT /api/auth/me` — Обновление данных пользователя

#### Бронирования
- `POST /api/bookings` — Создать бронирование (требует авторизации)
- `GET /api/bookings/my` — Мои бронирования (требует авторизации)
- `GET /api/bookings/{id}` — Информация о бронировании
- `DELETE /api/bookings/{id}` — Отменить бронирование
- `POST /api/bookings/check-availability` — Проверить доступность времени

#### Администрирование
- `GET /api/admin/bookings` — Все бронирования (для админов)
- `GET /api/admin/users` — Список пользователей
- `GET /api/admin/projects` — Управление проектами

### Пример запроса авторизации

```bash
curl -X POST "http://localhost:8000/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{
    "phone": "+79991234567",
    "code": "1111"
  }'
```

**Ответ:**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

## 💻 Использование

### Вход в систему

1. Откройте http://localhost:3000
2. Нажмите на иконку пользователя в шапке сайта
3. Введите номер телефона (например: `+79991234567`)
4. Введите код подтверждения: **1111** (для всех пользователей)
5. Нажмите "Войти"

### Создание бронирования

1. Войдите в систему (см. выше)
2. Нажмите кнопку "Записаться онлайн" на главной странице
3. Заполните форму:
   - Имя
   - Телефон
   - Выберите услугу
   - Выберите дату
   - Выберите время
   - (Опционально) Добавьте комментарий
4. Нажмите "Записаться"

**Важно:** Если выбранное время уже занято, вы получите уведомление и сможете выбрать другое время.

### Просмотр бронирований

1. Войдите в систему
2. Откройте профиль (иконка пользователя)
3. В разделе "Мои записи" вы увидите все ваши бронирования
4. Вы можете отменить бронирование, нажав соответствующую кнопку

### Административная панель

1. Войдите как администратор (пользователь с ролью `admin`)
2. Перейдите на http://localhost:3000/admin
3. Доступны разделы:
   - **Проекты** — управление портфолио
   - **Пользователи** — управление пользователями
   - **Роли** — управление ролями

## 🔧 Разработка

### Запуск в режиме разработки

**Backend:**
```bash
cd backend
python main.py
# Или с автоперезагрузкой:
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

**Frontend:**
```bash
npm run dev
```

### Сборка для production

**Frontend:**
```bash
npm run build
npm start
```

**Backend:**
```bash
# Используйте production ASGI сервер, например:
uvicorn main:app --host 0.0.0.0 --port 8000 --workers 4
```

### Тестирование

**Backend:**
```bash
cd backend
python test_api.py
```

**Frontend:**
```bash
npm run lint
```

### Переменные окружения

**Backend** (`.env` в папке `backend/`):
```env
DATABASE_URL=sqlite:///./hairlab.db
SECRET_KEY=your-secret-key-change-in-production
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

**Frontend** (`.env.local` в корне проекта):
```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```
