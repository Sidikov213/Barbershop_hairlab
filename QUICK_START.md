# 🚀 Hair Lab - Быстрый старт

## Что было сделано

✅ **Frontend (Next.js)**
- Вход по номеру телефона вместо email
- Код подтверждения "1111" вместо пароля
- Интеграция с бэкендом для бронирований

✅ **Backend (FastAPI + SQLite)**
- API для авторизации по телефону и коду
- Система бронирований с проверкой занятости времени
- База данных SQLite для хранения пользователей и бронирований
- JWT токены для безопасности
- Защита от двойного бронирования одного времени

## 📋 Требования

- Python 3.8+ (для бэкенда)
- Node.js 18+ (для фронтенда)

## ⚡ Быстрый запуск (Windows)

### Вариант 1: Автоматический запуск

Дважды кликните на файл `start-all.bat` в корне проекта. Он запустит и бэкенд, и фронтенд автоматически.

### Вариант 2: Ручной запуск

**Терминал 1 - Backend:**
```bash
cd backend
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
python init_db.py
python main.py
```

**Терминал 2 - Frontend:**
```bash
npm install
npm run dev
```

## 🌐 Адреса

- **Фронтенд**: http://localhost:3000
- **Бэкенд API**: http://localhost:8000
- **API Документация**: http://localhost:8000/docs

## 🔑 Как использовать

### 1. Вход в систему

1. Откройте http://localhost:3000
2. Нажмите иконку пользователя в шапке (справа)
3. Введите любой номер телефона (например: `+79991234567`)
4. Введите код: `1111`
5. Нажмите "Войти"

### 2. Создание бронирования

1. **Войдите в систему** (см. выше) - это обязательно!
2. Нажмите "Записаться онлайн"
3. Заполните форму:
   - Имя
   - Телефон
   - Выберите услугу
   - Выберите дату
   - Выберите время
4. Нажмите "Записаться"

⚠️ **Важно**: Если время уже занято другим пользователем, вы получите ошибку и сможете выбрать другое время.

## 🔧 API Endpoints

### Авторизация
- `POST /api/auth/login` - Вход (телефон + код)
- `GET /api/auth/me` - Информация о пользователе

### Бронирования
- `POST /api/bookings` - Создать бронирование (требует авторизации)
- `GET /api/bookings/my` - Мои бронирования (требует авторизации)
- `DELETE /api/bookings/{id}` - Отменить бронирование (требует авторизации)
- `POST /api/bookings/check-availability` - Проверить доступность времени

## ❗ Возможные проблемы

### "Для бронирования необходимо войти в аккаунт"
**Решение**: Нажмите иконку пользователя в шапке и войдите с кодом 1111

### "Это время уже занято"
**Решение**: Другой пользователь уже забронировал это время. Выберите другое.

### "Session expired"
**Решение**: Токен истёк (30 минут). Войдите заново.

### Backend не запускается
**Решение**: 
1. Проверьте что установлен Python 3.8+
2. Установите зависимости: `pip install -r requirements.txt`
3. Проверьте что порт 8000 свободен

### Frontend не подключается к Backend
**Решение**: Убедитесь что backend запущен на http://localhost:8000
