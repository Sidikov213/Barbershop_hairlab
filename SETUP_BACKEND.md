# Инструкция по запуску проекта Hair Lab

Этот проект состоит из двух частей:
- **Frontend** (Next.js) - находится в корне проекта
- **Backend** (FastAPI) - находится в папке `backend/`

## Требования

- Node.js 18+ (для фронтенда)
- Python 3.8+ (для бэкенда)

## Быстрый старт

### 1. Запуск Backend (FastAPI)

```bash
# Перейдите в папку backend
cd backend

# Создайте виртуальное окружение (рекомендуется)
python -m venv venv

# Активируйте виртуальное окружение
# Windows:
venv\Scripts\activate
# Linux/Mac:
# source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt

# (Опционально) Инициализируйте базу данных с тестовыми данными
python init_db.py

# Запустите сервер
python main.py
```

Backend будет доступен по адресу: **http://localhost:8000**

Документация API: **http://localhost:8000/docs**

### 2. Запуск Frontend (Next.js)

Откройте **новый терминал** (оставьте backend запущенным) и выполните:

```bash
# Вернитесь в корневую папку проекта
cd ..

# Установите зависимости (если ещё не установлены)
npm install

# Запустите dev сервер
npm run dev
```

Frontend будет доступен по адресу: **http://localhost:3000**

## Использование системы

### Вход в систему

1. Откройте сайт: http://localhost:3000
2. Нажмите кнопку входа (иконка пользователя в шапке)
3. Введите любой номер телефона (например: +79991234567)
4. Введите код: **1111** (это код для всех пользователей)
5. Нажмите "Войти"

### Создание бронирования

1. Войдите в систему (см. выше)
2. Нажмите кнопку "Записаться онлайн"
3. Заполните форму:
   - Имя
   - Телефон
   - Выберите услугу
   - Выберите дату
   - Выберите время
   - (Опционально) Добавьте комментарий
4. Нажмите "Записаться"

**Важно:** Если выбранное время уже занято другим пользователем, вы получите сообщение об ошибке и сможете выбрать другое время.

## API Endpoints

### Авторизация

- `POST /api/auth/login` - Вход по телефону и коду
- `GET /api/auth/me` - Получить информацию о текущем пользователе

### Бронирования

- `POST /api/bookings` - Создать бронирование (требует авторизации)
- `GET /api/bookings/my` - Получить мои бронирования (требует авторизации)
- `DELETE /api/bookings/{id}` - Отменить бронирование (требует авторизации)
- `POST /api/bookings/check-availability` - Проверить доступность времени

### Администрирование

- `GET /api/admin/bookings` - Получить все бронирования

## База данных

Проект использует SQLite. База данных автоматически создаётся в файле `backend/hairlab.db` при первом запуске.

### Структура данных

**Пользователи (users):**
- id - уникальный идентификатор
- phone - номер телефона (уникальный)
- name - имя пользователя
- created_at - дата создания
- is_active - активен ли пользователь

**Бронирования (bookings):**
- id - уникальный идентификатор
- user_id - ID пользователя
- service - название услуги
- date - дата (YYYY-MM-DD)
- time - время (HH:MM)
- comment - комментарий
- status - статус (confirmed/cancelled/completed)
- created_at - дата создания

## Возможные проблемы

### Backend не запускается

1. Убедитесь, что установлен Python 3.8+
2. Проверьте, что все зависимости установлены: `pip install -r requirements.txt`
3. Проверьте, что порт 8000 не занят другим приложением

### Frontend не подключается к Backend

1. Убедитесь, что backend запущен на http://localhost:8000
2. Проверьте CORS настройки в `backend/main.py`
3. Откройте консоль браузера (F12) для просмотра ошибок

### Ошибка "Session expired"

Токен истёк. Просто войдите снова через форму входа.

### Время уже занято

Другой пользователь уже забронировал это время. Выберите другое свободное время.

│   ├── main.py          # Главный файл приложения
│   ├── models.py        # Модели базы данных

### Hot Reload

- **Frontend**: автоматическая перезагрузка при изменении файлов
- **Backend**: автоматическая перезагрузка включена (`reload=True` в uvicorn)

## Поддержка

При возникновении проблем:

1. Проверьте логи в терминале
2. Откройте http://localhost:8000/docs для тестирования API
3. Проверьте консоль браузера (F12) для ошибок фронтенда
4. Убедитесь, что оба сервера (frontend и backend) запущены

## Производство

Для развёртывания на production:

1. **Backend**: 
   - Измените `SECRET_KEY` в config.py
   - Используйте PostgreSQL вместо SQLite
   - Настройте HTTPS
   - Настройте CORS для вашего домена

2. **Frontend**:
   - Создайте production build: `npm run build`
   - Настройте переменную окружения `NEXT_PUBLIC_API_BASE_URL`
   - Разверните на Vercel, Netlify или другом хостинге
