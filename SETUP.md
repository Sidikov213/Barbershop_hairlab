# 🚀 Инструкция по запуску проекта

## Требования

- **Node.js** версии 18.x или выше
- **npm** или **yarn** (или **pnpm**)

## Шаги для запуска

### 1. Установка зависимостей

```bash
npm install
```

или

```bash
yarn install
```

### 2. Настройка переменных окружения

Создайте файл `.env.local` в корне проекта со следующим содержимым:

```env
# URL вашего бэкенд API сервера
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
```

**Важно:** 
- Если ваш бэкенд работает на другом порту или адресе, измените значение `NEXT_PUBLIC_API_BASE_URL`
- Если бэкенд не запущен, приложение будет использовать fallback на mock-данные для некоторых страниц

### 3. Запуск проекта в режиме разработки

```bash
npm run dev
```

или

```bash
yarn dev
```

Приложение будет доступно по адресу: **http://localhost:3000**

### 4. Сборка для production

```bash
npm run build
npm start
```

### 5. Проверка кода (линтер)

```bash
npm run lint
```

## 📁 Структура проекта

- **`/src/app`** - страницы Next.js (App Router)
  - `/` - главная страница
  - `/projects/[id]` - страница проекта
  - `/admin` - админ-панель
  - `/auth/login` - страница входа
- **`/src/components`** - React компоненты
- **`/src/lib`** - утилиты и API клиент
- **`/public`** - статические файлы (изображения, иконки)

## 🌐 Доступные маршруты

- **Главная страница:** http://localhost:3000
- **Админ-панель:** http://localhost:3000/admin
- **Вход в систему:** http://localhost:3000/auth/login
- **Страница проекта:** http://localhost:3000/projects/1

## ⚠️ Важные замечания

1. **Бэкенд API:** Убедитесь, что ваш бэкенд сервер запущен и доступен по адресу, указанному в `NEXT_PUBLIC_API_BASE_URL`
2. **Переменные окружения:** Файл `.env.local` должен быть в корне проекта и не должен попадать в Git (уже добавлен в `.gitignore`)
3. **Порт:** По умолчанию Next.js использует порт 3000. Если он занят, Next.js автоматически предложит использовать другой порт

## 🐳 Запуск через Docker

Если у вас установлен Docker:

```bash
docker build -t soft-studio-frontend .
docker run -p 3000:3000 -e NEXT_PUBLIC_API_BASE_URL=http://localhost:8000 soft-studio-frontend
```

## 🔧 Решение проблем

### Ошибка "npm не является внутренней или внешней командой"
Установите Node.js с официального сайта: https://nodejs.org/

### Ошибка "выполнение сценариев отключено в этой системе" в PowerShell

Эта ошибка возникает из-за политики выполнения скриптов в Windows PowerShell. Есть несколько способов решения:

#### Вариант 1: Использовать CMD (Командная строка) вместо PowerShell
Откройте **CMD** (Win+R → введите `cmd` → Enter) и выполните команды там.

#### Вариант 2: Изменить Execution Policy (требует прав администратора)
1. Откройте PowerShell **от имени администратора** (правой кнопкой → "Запуск от имени администратора")
2. Выполните команду:
   ```powershell
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```
3. Подтвердите действие (введите `Y`)

#### Вариант 3: Временно обойти политику
Для одной сессии PowerShell выполните:
```powershell
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
```

#### Вариант 4: Использовать Git Bash или другой терминал
Если установлен Git, используйте Git Bash, где npm работает без проблем.

### Порт 3000 уже занят
Next.js автоматически предложит использовать другой порт, или вы можете указать порт вручную:
```bash
npm run dev -- -p 3001
```

### Ошибки при подключении к API
Проверьте:
- Запущен ли бэкенд сервер
- Правильно ли указан `NEXT_PUBLIC_API_BASE_URL` в `.env.local`
- Доступен ли бэкенд из браузера (проверьте CORS настройки)
